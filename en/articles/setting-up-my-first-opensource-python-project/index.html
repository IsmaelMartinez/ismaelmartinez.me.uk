    Project Overview

    The project is called logger-to-kibana, and it centres on automation principles. The tool parses loggers in code and generates Kibana visualizations from log messages encountered during execution.

    Key Features

    The project includes three main commands:

    • process ‚Äî processes a folder
    • process_and_generate ‚Äî processes a folder and generates visualizations
    • process_generate_and_send ‚Äî executes full pipeline including transmission to Kibana

    Installation

    Users can install dependencies via:

    pip install -r requirements.txt

    And access help documentation by running:

    python main.py

    Configuration

    Default settings exist in settings.ini, with customisation available through the LOGGER_TO_KIBANA_CONFIG environment variable.

    This allows you to configure:

    • The Kibana endpoint URL
    • Authentication credentials
    • Visualization templates and defaults
    • Log parsing patterns
