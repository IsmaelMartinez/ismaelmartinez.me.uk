Este es el inicio de una serie que documenta la creación de mi primer proyecto en PyPI. Reconozco que esto representa tanto mi primera contribución al código abierto como mi primera incursión en Python, por lo que hay un margen considerable de mejora.

## Visión General del Proyecto

El proyecto se llama **logger-to-kibana** y se centra en principios de automatización. La herramienta analiza los loggers en el código y genera visualizaciones de Kibana a partir de los mensajes de log encontrados durante la ejecución.

## Características Principales

El proyecto incluye tres comandos principales:

- `process` -- procesa una carpeta
- `process_and_generate` -- procesa una carpeta y genera visualizaciones
- `process_generate_and_send` -- ejecuta el pipeline completo incluyendo la transmisión a Kibana

## Instalación

Los usuarios pueden instalar las dependencias mediante:

```bash
pip install -r requirements.txt
```

Y acceder a la documentación de ayuda ejecutando:

```bash
python main.py
```

## Configuración

Existen ajustes predeterminados en `settings.ini`, con la posibilidad de personalización a través de la variable de entorno `LOGGER_TO_KIBANA_CONFIG`.

Esto te permite configurar:

- La URL del endpoint de Kibana
- Credenciales de autenticación
- Plantillas y valores predeterminados de visualización
- Patrones de análisis de logs
